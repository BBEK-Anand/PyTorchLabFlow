The Research Workflow is Inherently Chaotic

The path from hypothesis to publication is non-linear. Standard tools, built for production, often fail to support the exploratory process.

The Experimentation Mess

In early-stage research, we juggle multiple variants of models, datasets, and optimizers. This leads to an explosion of scripts and configurations before any "version 1" is even established. It’s frustrating and time-consuming to manage configs, track artifacts, and ensure results are reproducible—a challenge highlighted by the ongoing reproducibility crisis in AI.

A Method for the Mess

PyTorchLabFlow imposes a clean structure on top of the chaos with two core concepts.

1. The Component: Reusable Building Blocks

The `Component` is the fundamental unit: a self-contained, reusable piece of code (model, dataset, etc.). Defined by its location and arguments in a config, it promotes modularity and eliminates duplicated code.

Nesting is supported: build complex, modular systems from simple, swappable parts without touching your Python scripts.

2. The Pipeline: The Reproducible Experiment

The `PipeLine` class assembles your Components into a complete experiment via a single YAML config. Each `PipeLine` is given a unique ID, capturing the exact state of your code and parameters. This makes every run fully traceable and perfectly reproducible.

Installation & Quickstart

No servers. No databases. No boilerplate.

1. Install

pip install pytorchlabflow

2. Configure

# config.yaml
model:
  loc: my_project.models.MyCNN
  args: { num_classes: 10 }

3. Run

from pytorchlabflow import PipeLine

p = PipeLine()
p.new("exp01", "config.yaml")
p.run(your_train_func)
